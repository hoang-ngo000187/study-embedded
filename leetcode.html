02. Contains Duplicates (LeetCode #217)

Hoang Ngo June 05, 2026

1. Hiểu rõ vấn đề cần giải quyết

You are given an array of integers, nums. Your task is to determine whether any value appears at least twice in the array. In other words, you should return true if there are any duplicate elements, and false otherwise.

Example:

Input: nums = [1, 2, 3, 1]
→ Output: true (because the number 1 appears twice)

Input: nums = [1, 2, 3, 4]
→ Output: false (because no element appears more than once)

The challenge is to find a solution that works efficiently with large input arrays.

2. Brute Force Approach

Cách đơn giản nhất để giải quyết vấn đề này là so sánh từng phần tử với mọi phần tử khác trong mảng. Phương pháp vét cạn này hoạt động được, nhưng nó có một nhược điểm đáng kể về hiệu quả.

bool containsDuplicate(vector& nums) { for (int i = 0; i < nums.size(); i++) { for (int j = i + 1; j < nums.size(); j++) { if (nums[i] == nums[j]) { return true; } } } return false; }

Trong giải pháp này, chúng ta sử dụng hai vòng lặp lồng nhau để so sánh từng phần tử với mọi phần tử khác, dẫn đến độ phức tạp thời gian là O(n2). Mặc dù phương pháp này sẽ cho ra kết quả chính xác, nhưng nó không phù hợp với các tập dữ liệu lớn vì mất quá nhiều thời gian để xử lý.

3. Efficient Solution - I: Sắp xếp trước rồi kiểm tra

Ý tưởng: Nếu có các phần tử trùng nhau thì sau khi sắp xếp chúng sẽ nằm cạnh nhau.

Ví dụ:

[1,2,4,2,7,6,3]

Sau khi sort:

[1,2,2,3,4,6,7]

Ta chỉ cần kiểm tra:

nums[i] == nums[i + 1]

Nếu đúng ⇒ có phần tử trùng.

sort(nums.begin(), nums.end()); for (int i = 0; i < n - 1; i++) { if (nums[i] == nums[i + 1]) return true; } return false;

Độ phức tạp

  • Sort: O(n log n)
  • Duyệt mảng: O(n)

Tổng: O(n log n)

Space: thường O(1) hoặc phụ thuộc thuật toán sort.

4. Efficient Solution - II: Hash Set (Tối ưu nhất)

Ý tưởng:

  • Dùng một Hash Set để lưu các phần tử đã gặp.
  • Khi duyệt từng phần tử:
  • Nếu phần tử đã tồn tại trong Set ⇒ trùng ⇒ trả về true.
  • Nếu chưa tồn tại ⇒ thêm vào Set.
class Solution { public: bool containsDuplicate(vector& nums) { unordered_set s; for (int num : nums) { if (s.find(num) != s.end()) { return true; } s.insert(num); } return false; } };

Time and Space Complexity

  • Time Complexity: Độ phức tạp về thời gian của giải pháp hiệu quả là O(n), trong đó n là số phần tử trong mảng. Chúng ta lặp qua danh sách một lần, và độ phức tạp thời gian trung bình để chèn và kiểm tra sự hiện diện trong hash set là O(1).
  • Space Complexity: Độ phức tạp về không gian cũng là O(n), bởi vì, trong trường hợp xấu nhất, chúng ta có thể cần lưu trữ tất cả n phần tử trong tập hợp nếu không có phần tử trùng lặp.
  • peek: giống pop nhưng không lấy phần tử ra.
  • empty: hàng đợi rỗng khi cả hai ngăn xếp đều trống.

    • Vì mỗi phần tử chỉ được chuyển giữa hai stack tối đa một lần, nên mỗi thao tác có thời gian trung bình amortized là O(1). Đây là điểm quan trọng để hiểu vì sao giải pháp vẫn hiệu quả dù có bước dịch chuyển toàn bộ stack.
